% TODO: file is supposed to cover the whole original set of Otoman Turkish
% with no support for EC encoding; needs no be done first
%
% Original patterns need special fonts to be able to use them,
% and use several tricks that only enable same hacks to work properly.
% 
% I did not dive into details of conversion, but modern Turkish only uses:
% - 4 "special" vowels: ıiöü
% - 3 additional consonants: çğş
% while the converted patterns also use:
% - acircumflex: â
% - ocircumflex: ô
% - ntilde: ñ
% One could argue that these are used just because they can,
% but on the other hand there is no zdotaccent (ż).
% 
% The conversion misencoded dotlessi into ^^11 (should be ^^19).
% 
% 2008: patterns converted into Unicode.
% Several characters could be added or removed, but I (Mojca) could not play the judge
% and did not do any changes, even though I originally intended to fix turk_hyf.c.
% 
% -------
% Original encoding:
% 
% a
% a=  â
% e
% i   ı
% i=  î
% i:  i
% o
% o=  ô
% o:  ö
% u
% u:  ü
% 
% -------
% 
% character / UTF-8 letter / EC code / Unicode / Unicode name
% 
% @      -- arabic hamza
% #      -- ayn

% c

% d
% d!  ḍ  -- U+1E0D   LATIN SMALL LETTER D WITH DOT BELOW
% d=  ḏ  -- U+1E0F   LATIN SMALL LETTER D WITH LINE BELOW
% f
% g

% g:  ġ  -- U+0121   LATIN SMALL LETTER G WITH DOT ABOVE
% h

% h=  ẖ  -- U+1E96   LATIN SMALL LETTER H WITH LINE BELOW
% j
% k
% k!  ḳ  -- U+1E33   LATIN SMALL LETTER K WITH DOT BELOW
% l
% m
% n
% n=  ñ  F1 U+00F1   LATIN SMALL LETTER N WITH TILDE
% p
% r
% s

% s=     --    s with line below - not even in unicode

% t

% y
